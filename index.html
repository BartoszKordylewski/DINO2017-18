Komenda Oryginał Opis Przykład
git commit -a -m "" git commit- Służy do zapisania/zatwierdzenia
bieżacego stanu projektu.		 Komenda do opisania można podzielić na dwie części,
które po połączeniu wykonają jedną i drugą czynność.

"git commit -a"- ta część odpowiada za zatwierdzenie wszystkich
plików dodanych poprzez "git add" oraz zatwierdza
wszystkie pliki dodane od tego czasu.

"git commit -m "..." "- jest odpowiedzialny za dodanie "commit message" do zatwierdzenia.		 git commit -a -m "Commit message"
git add git add Komenda jest odpowiedzialna za dodawanie plików.
I. Można dodać określony plik.
II. Można dodać wszystkie od razu 		I. git add < nazwa_pliku >
II. git add *
git checkout "branch" git checkout "" Komenda przełącza z jednej "gałęzi" na drugą, która nazywa się "branch" git checkout "branchname"
git remote add "remote_name" "git url" git remote add "name" "server" Jeśli lokalne repozytorium nie zostało podłaczone do zdalnego serwera,
to można to zrobić za pomocą tej komendy.
"remote name"- nazwa zdalnego serwera.
"git url"- link do serwera. 		git remote add "name" ""nazwa_uzytkownika"@"domena_serwera""
git blame -C "filename" git blame -function Jest to komenda dzięki której możemy sprawdzić,
kiedy dokonano pewnych zmian,
dzięki użyciu funkcji "-C" możemy podać nazwę pliku,
który chcemy sprawdzić. 		git blame -C "Test"
git branch "name" git branch Dzięki tej komendzie dowiemy się o wszystkich gałęziach
w naszym repozytorium i wskaże nam gdzie aktulanie się znajdujemy. 		git branch "Projekt"
