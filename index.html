git config --global user.name "John Doe"
Komenda służąca do ustawienia swojej nazwy użytkownika.
Ogólnie polecenie git config służy do ustawiania niestandardowych konfiguracji systemu.
Git korzysta z zestawu plików konfiguracyjnych, które służą do zdefiniowania ustawień dla określonych poziomów systemu.
Użycie opcji git config --system powoduje czytanie lub edytowanie konfiguracji z pliku /etc/gitconfig, która stosuje się do każdego użytkownika i repozytorium w systemie.
Opcja git config --global służy do czytania lub edytowania konfiguracji z pliku ~/.gitconfig lub ~/.config/git/config, który zawiera konfigurację specyficzną dla danego użytkownika.
Opcja git config --local czyta lub edytuje konfigurację z pliku .git/config, należącego do aktualnego repozytorium, określającego ustawienia specyficzne tylko dla niego. Jest to domyślny poziom konfiguracji, w przypadku braku określenia odpowiedniej opcji w poleceniu git config.
Ustawienia każdego z poziomów nadpisują te, które są określone na niższym poziomie, zgodnie z kolejnością: system, global, local, gdzie global nadpisuje system, a local wszytkie pozostałe.

git commit -a -m ""
Records changes to the repository.
-a - also stages files (modified and deleted, but not new) (works almost like git add -u) -m "" - with commit message inside ""

git add . - dodaje wszystkie pliki do śledzenia git checkout "branch" - przełączanie do nowej gałęzi (komenda wykona się poprawnie jedynie, gdy nowa gałąź została wcześniej utworzona za pomocą git branch "name")
git branch "name" - tworzy nową gałąź o podanej nazwie

git reset HEAD
git reset to przeciwieństwo git add. Komenda służąca do cofania zmian w lokalnym repozytorium.
HEAD oznacza historię commitów (Commit History), która jest częścią jednego z trzech drzew, na których operuje git reset.

git -C blame - pozwala sprawdzic, kto jest autorem danej linii kodu, 'obwinic tworce'

git log --graph --all

git fetch "remote" "branch"

Pobiera wszystkie zmiany z remote'a o nazwie remote z gałęzi branch.

git remote add "remote_name" "git url"

Komenda służy do dodawania zdalnych repozytoriów pod skrótem. Dzięki temu będziemy mogli z łatwością odnosić się w przyszłości. W miejsce "remote name" wpsiujemy skrót pod jakim ma zostać zapisane dodatkowe repzytorium. W polu "git url" umieszczamy link do tego repozytorium. Przy pomocy tego następnym razem gdy będziemy chcieli np. zrobić fetch od naszego współpracownika wystarczy, że użyjemy fetch z parametrem, który wpisaliśmy w "remote name".

Komendy Git GRUPA CF
git config --global color.ui true
Git może automatyczie pokazywać w kolorze większość wyników swojego działania. Można bardzo dokładnie określić co ma być pokazane w kolorze, oraz w jakis sposób.
Color.iu ustawione na "true" włącza kolorowanie wyników dzałania rodzin komend git w konsolach które to wspierają. Jeśli output jest przekierowany do pliku, to nie zostaną dodane sekwencje ANSI mówiące o kolorze.
Inne możliwe ustawienia Color.io to "false" i "always". Git domyślnie koloruje składnie. Przełącznik true włącza ten feature, false wyłącza.
git commit "filename"
Commituje zmianę tylko pliku o nazwie "filename", zachowując w staging wcześniej dodane pliki za pomocą 'git add'.
przykład: git commit "index.html". Po wykonaniu takiej komendy zostanie przesłany tylko plik index.html. Uruchomi się edytor tekstu w celu utworzenia opisu zmian. Komentarz można również utworzyć w terminalu przy pomocy atrubuty -m (git commit "index.html" -m "tresc opisu zmian") co przy niewielkich wiadomościach przyspierza proces comitowania. Commituje zmianę tylko pliku o nazwie "filename", zachowując w staging wcześniej dodane pliki za pomocą 'git add'. Szczególnie przydatne gdy dokonaliśmy zmian w wielu plikach projektu, a commitować chcemy tylko jeden.
git branch "name"
Tworzy nową gałąź o nazwie "name", ale do niej nie przechodzi, trzeba to zrobić za pomocą 'git checkout "name"'.
git branch -v
W trybie listy wyświetla skrót sha1 i temat commita w jednej linii dla każdej sekcji head, ze statusem względem nielokalnej gałęzi.
git push "remote" "branch"
Synchronizacja z serwerem zdalnym "remote" gałęzi "branch". Inaczej wypycha gałąź branch na server remote. Ogólna postać komendy: git push [nazwa zdalnego repozytorium] [nazwa gałęzi] Przydatna gdy chcemy udostępnić dokonane przez nas zmiany członkom zespołu, którzy pracują przy tym samym projekcie.
git log "branch" --not "branch"
Słuzy do wyświetlenia listy commitów z gałęzi „branch” z wyjątkiem w postaci gałęzi „branch”. W praktyce, nie wyświetla nic, gdyż ma wyświetlić listę commitów z gałęzi "branch" ale z wyjątkiem gałęzi o nazwie "branch", więc w sumie żadna gałąź nie jest podana jako argument. Samej komendy git log używamy by spojrzeć w historię projektu i sprawdzić dokonane zmiany.
git blame -C "filename"
Rozpoznaje linie w pliku "filename" skopiowane z innych plików, które były zmodyfikowane w ramach jednego commita. Przydatne przy reorganizacji struktury programu.
git bisect (start, bad, good, reset)
Służy do wyszukiwania bugów w kodzie na podstawie commitów, złego (bad - obecnego lub innego commita w którym bug występuje) dobrego (good - commita kodu w ktorym bug nie występuje). Można wykorzystać nie tylko do wyszukiwania bugów ale i do obserwowania kiedy następiła jakaś zmiana w kodzie. "Start" służy jako rozpoczecie sekwencji wyszukiwania zmiany. Pierwszym krokiem jest wskazanie commita mniej więcej w połowie pomiędzy good i bad i przywrocenie kodu do stanu z tego commita. Jeżeli to w nim znajduje się błąd to oznaczamy go jako bad z użyciem 'git bisect bad', a jeżeli nie wystepuje to oznaczamy jako 'git bisect good' i ponownie otrzymujemy commita w którym możemy mieć tego buga. Proces powtarzamy do znalezienia commita który wprowadza bug/zmiany. Reset to powrót do stanu z przed bisekcji, czyli powrót do oryginalnego head.
gitk -all
gitk – Przeglądarka repozytorium na gicie. gitk -all – Wyświetla wszystkie branche, tagi, etc. Przydatne gdy chcemy wyswietlic całą zawartość naszego repozytorium.
git merge upstream/master
Polecenie zacznie od próby aktualizowania, jeśli jakieś zmiany zostały przeprowadzone. W przeciwnym wypadku wyświetlony zostanie komunikat, że wszystie pliki są "Up-to-date".
