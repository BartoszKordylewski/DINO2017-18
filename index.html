GitHub INO 4.2

  • Grupa CE

  • git log --graph --all - Polecenie służy konfiguracji adresu e-mail konta Git.

    Polecenia warto użyć od razu po instalacji Git Jest to ważne, ponieważ każda operacja zatwierdzenia w Git korzysta z tych informacji, które stają się integralną częścią zatwierdzeń przesyłanych i pobieranych później do i z serwera.

  • git commit -a -m "" - Polecenie służy do zatwierdzenia zmian w projekcie. Opcja -a dokonuje zatwierdzenia zmian podanych plików automatycznie

    Jak wiemy zmiany w projekcie musimy zatwierdzić. Po dodaniu do składni polecenia git commit opcji -a każdy zmieniony plik, który jest już śledzony, automatycznie trafi do poczekalni, dzięki czemu pominiesz część git add.

  • git diff --staged - Polecenie porównuje zmiany z poczekalni z ostatnią migawką

    Polecenie to jest bardzo ważne. Używając Gita chcemy kontrolować kod. Dzięki porównaniu zaktualizowanej wersji z poprzednią mamy większą kontrolę nad kodem, a tą umożliwa nam to polecenie

  • git checkout -b "branch" - Polecenie tworzy nową gałąź w obecnym repozytorium o nazwie podanej w cudzysłowiu i przełącza się na nią

    Komendę wykorzystujemy gdy chcemy wprowadzić pewne zmany w kodzie, a niekoniecznie pracować w obecnej gałęzi. Polecenie możemy również użyć dodając nową osobę do repozytorium, tak aby mogła ona od razu zacząć pracować na własnej gałęzi

  • git log --graph --all - Polecenie wyświetlające historię naszego repozytorium w postacji grafu

    Polecenia używamy gdy chcemy zobaczyć w formie tekstowej historie Committów, obejrzeć zmiany itp.

  • git branch -d "branch" - Polecenie usuwa gałąź o nazwie podanej w cudzysłowiu

    Polecenia używamy gdy chcemy usunąć daną gałąź z repozytorium. Możemy go użyć gdy chcemy wykluczyć część kodu napisanego w danej gałęzi przez jednego z użytkowników

  • git remote add "remote_name" "git url" - Polecenie służy do dodania zdalnego repozytorium, którego nazwę podajemy w pierwszym cudzysłowiu, a URL do niego w drugim

    Praca przy dużym projekcie wymaga użycia zdalnego repozytorium. Komendy używamy w celu współpracy w grupie, zarządzania zdalnym projektem itp.

  • git fetch "remote" "branch" - Polecenie służy do uzyskiwania danych ze zdalnych repozytorium, podajemy nazwę zdalnego repozytorium oraz gałąź

    Polecenia możemy użyć gdy pracujemy nad dużym projektem w kilka osób. Polecenie pozwoli nam na uzyskanie potrzebnych nam danych ze zdalnego repozytorium
